# DeepThink v1.0.7 发布说明

> 自托管的多用户本地 AI Agent Loop Engineering 系统（支持桌面端 + 浏览器 + 移动端）
> Powered By AI Genius Institute，AI 光剑

发布日期：2026-07-19

v1.0.7 是一个**多引擎可用性修复版本**，集中修复 Codex 与 OpenCode 两个非 Claude 引擎在真实使用中暴露的链路缺陷，使三大引擎（Claude / Codex / OpenCode）都能真正完成"用户提问 → 模型回答"的闭环。本版本不引入新功能，聚焦于让已上线的引擎从"能跑起来"到"能正确回答"。

---

## 🐛 缺陷修复

### OpenCode 引擎（重点）

v1.0.6 的 OpenCode 引擎在端到端验证中无法正确回答——任何提问要么报错无输出、要么把用户输入原样复读。本版本定位并修复了**三个叠加根因**，使 OpenCode 引擎真正可用，并兼容 DeepThink 统一 skill 层：

- **修复 SubAgent 回复复读用户输入且每轮注入 `<system_context>`**：`processAgentConversation` 对所有引擎统一用 `getSession()`（读 `session_id` 列）取会话，但 OpenCode/Codex/AtomCode 的会话存在各自专用列，读错列导致 sessionId 恒为 `undefined` → 每轮判为新会话 → 每轮注入历史 + 每轮新建 OpenCode 会话丢上下文。新增 `resolveAgentSessionId()` 按引擎读对应列（与已有"写 session 按引擎分流"模式对齐）。同时 `opencode-engine.ts` 的 `runOneTurn` 把所有 `message.part.updated` 文本 part 累加进 `fullText`，但 OpenCode 也会为用户消息的 TextPart（含注入的 system_context）推送该事件，且 part 无 role 字段，导致回复 = 整段 prompt + 用户问题复读。改为通过 `message.updated` 的 `info.role==='user'` 收集 `userMessageIds`，跳过属于用户消息的 part。（`d99be5a`）

- **修复 provider 配置字段名错误导致 LLM 根本未被调用**：`writeOpencodeConfigFile` 原写顶层蛇形 `api_key`/`base_url` 且漏 `api` 字段，但 OpenCode v1 provider schema 要求 `options.apiKey`/`options.baseURL`（驼峰、在 `options` 下）+ 顶层 `api`（API 类型）。字段名不对 → OpenCode 读不到 baseURL → 解析成 `undefined/chat/completions cannot be parsed as a URL` → LLM 从未被调用（此前"echo 回复"其实只是用户输入被原样复读，无真实模型输出）。改为 `api: "openai"`（DeepThink 的 opencode provider 均为 OpenAI 兼容 baseURL，如 dashscope `/compatible-mode/v1`）+ `options.apiKey`/`options.baseURL`。依据 OpenCode 源码 `packages/core/src/v1/config/provider.ts` 的 `Info` schema。（`99ce685`）

- **OpenCode 源码侧：skill 解析失败不再中断整段会话**（兼容 DeepThink skill 清单）：OpenCode serve 自动扫描宿主 `~/.claude/skills/**/SKILL.md`（Claude Code skill 兼容），单个断链/不可解析的 SKILL.md 会让 `add()` publish 一条 `session.error`，而 DeepThink engine 把任何 `session.error` 当致命错误 → 整段会话报错无输出。直接修改 OpenCode 源码 `packages/opencode/src/skill/index.ts` 的 `add()`：解析失败时只 log 跳过、不再 publish `session.error`，其余 skill 照常加载。重新打包二进制替换 `/home/me/.local/bin/opencode`（原 178MB 二进制备份为 `opencode.bak.20260719`，源码改动 commit 到本地 `dev` 分支防 `opencode upgrade` 丢失）。**不禁用** skill 扫描，实测 82 个 DeepThink skill 全部加载（`init count=82`），断链 officecli 仅 log 跳过、SSE 无 skill `session.error`。

详见 `docs/issues/2026-07-19-opencode-engine-echo-and-session.md` 与 `docs/issues/2026-07-19-opencode-skill-parse-crash.md`。

### Codex 引擎

- **修复 `agent_message` 文本丢失导致返回空回复**：codex-engine 的 `item.completed` 分支不处理 `agent_message`，而 qwen3.7-max 经 DashScope responses API 只以 `item.completed` 形式发出 `agent_message`（无 started/updated），导致模型答案文本被丢弃，`fullText` 为空 → "(Codex 返回空回复)"。在 `item.completed` 补 `agent_message` 处理，复用 `lastItemText` 增量去重。（`5b20bec`）
- **修复 `config.toml` env_vars 类型错误导致 `codex exited with code 1`**：codex `config.toml` 的 env_vars 类型校验错误，导致 codex 进程启动即退出。（`42dcea4`）
- **主机模式 workingDir 守卫**：主机模式 Codex/OpenCode workingDir 守卫，修复 `spawn ENOENT`。（`7397b90`）
- **ESM 下 `__dirname` 未定义**：Codex/OpenCode 引擎 ESM 下 `__dirname` 未定义导致对话无响应，改用 `import.meta.url` 解析。（`6f59940`）
